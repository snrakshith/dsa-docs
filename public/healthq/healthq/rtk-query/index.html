Fetching data in healthq via RTK Query#

Overview#

RTK Query is a powerful data-fetching and caching tool included in Redux Toolkit. It separates data management concerns from state management, streamlining the way developers handle server-side data in React applications.

Key Features#

  • Integrated with Redux Toolkit: Built on top of Redux, it uses createSlice and createAsyncThunk.
  • Auto-generated Hooks: Simplifies component usage by automatically generating hooks for defined endpoints.
  • Cache Management: Offers fine-grained control over cache lifecycles, including automated refetching and cache invalidation.
  • TypeScript Support: Fully written in TypeScript, providing strong type safety.

Summary#

  • POST: Use createItem to add new items.

  • DELETE: Use deleteItem to remove items.

Best Practices#

  • Single API Slice: Aim for one API slice per base URL to maintain clarity and organization.
  • Split Endpoints: modularize your code while keeping a unified API slice.
  • Error Handling: Always implement error handling in your components to enhance user experience.
