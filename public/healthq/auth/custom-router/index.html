Routing in the Web App#

This section describes how routing works in our web app using React Router with authentication handled via react-oidc-context. We ensure that only authenticated users can access private routes, and we handle loading states, sign-in redirection, and error pages.

Code Overview#

Key Concepts Used#

  1. react-router-dom for Routing:

    • <Routes> and <Route> components define the routes and their corresponding components.
    • <Navigate> helps redirect users based on authentication status.
    • Lazy Loading with React’s Suspense: Loads private routes asynchronously to improve performance.

  2. Authentication Handling via react-oidc-context:

    • useAuth hook: Manages the current user session and authentication state.
    • If the user is not authenticated, we trigger a sign-in redirect using auth.signinRedirect().

Understanding the CustomRoutes Component#

Detailed Walkthrough#

  1. Authentication Effect

    • useEffect checks whether the user is authenticated when the component loads.
    • If the user is not authenticated and no sign-in attempt has been made, the app triggers auth.signinRedirect().
    • setHasTriedSignin(true) ensures we don’t redirect multiple times.

  2. Loading State Management

    • If auth.isLoading is true, we show the <SplashScreen> component to indicate that the authentication process is in progress.

  3. Routes Configuration

    • If the user is authenticated, we navigate them from / to the dashboard.
    • Private routes (those requiring authentication) are wrapped inside PrivateRoutes, which are lazy-loaded using React.lazy.
    • routesConfig dynamically maps the route paths to components.

  4. Fallback for Undefined Routes

    • If a user navigates to a route that doesn’t exist, they are shown the ErrorsPage component via the * route.

Explanation of Key Components#

  1. PrivateRoutes:

    • This component holds the authenticated-only routes. Lazy-loading improves performance by loading them only when necessary.

  2. SplashScreen:

    • Displayed while the app checks the user’s authentication state or during lazy-loading.

  3. routesConfig:

    • An array of route definitions (path and component) imported from routesConfig.js.

Potential Issues and Debugging Tips#

  • Issue: App keeps redirecting to the sign-in page.
    Solution:

    • Ensure that the OIDC provider URL and client_id are correctly configured in your environment variables.
    • Verify that auth.isAuthenticated is being properly updated after login.

  • Issue: Routes are not rendering after login.
    Solution:

    • Check the paths in routesConfig and ensure they are valid.
    • Make sure PrivateRoutes is correctly defined and loaded.
